创建新项目过程

1.$ hexo new post 项目名称

​ eg:$ hexo new post 用递归方程求时间复杂度

​ 这条命令会在”source/_posts”目录下创建一个名为”用递归方程求时间复杂度.md”的Markdown文件，用于撰写新文章。接下来，在文本编辑器中打开该文件，开始撰写文章。

​ 接下来，在文本编辑器中打开该文件，开始撰写文章。你可以使用Markdown语法来格式化文章内容，并在其中介绍递归方程和时间复杂度的概念，以及如何使用递归方程来求解时间复杂度。

2.$ hexo clean && hexo s

​ 运行hexo clean && hexo s意味着先清除Hexo项目中的缓存文件，然后立即启动本地服务器以用于预览博客。

​ 一般来说，当你在Hexo博客中写完一篇文章后，需要先保存并关闭这个文件，然后再在终端窗口中执行hexo clean && hexo s命令来清除Hexo缓存文件并启动本地服务器，从而可以在浏览器中预览你刚刚写完的这篇文章。

​ 如果你在写作期间需要频繁地查看文章预览效果，也可以将终端窗口保持打开状态并使用刷新功能来更新预览效果，但要注意在每次更改文章内容后，需要保存并关闭文件，并等待Hexo自动重新编译生成，然后才能看到最新的预览效果。

总之，在写作过程中不需要运行hexo clean && hexo s命令，只有在你完成了一篇文章并准备进行预览时才需要运行该命令。

3.$ hexo d

​ 这个命令的作用是将已经生成好的静态网站发布到远程服务器上，以便让其他人也能够在互联网上访问到你的博客。

​ 具体来说，执行hexo d命令会按照配置文件中的相关设置，将已经生成好的网站代码上传到远程服务器中指定的路径中。需要注意的是，在执行该命令之前，你需要先在配置文件中正确地设置好服务器相关的参数，包括服务器地址、用户名、密码等信息。

​ 如果你已经设置好了这些参数并成功登录到服务器中，那么执行hexo d命令就可以将你的博客发布到互联网上了。发布成功后，其他人就可以通过浏览器访问你的博客了。

4.$ hexo s

​ 执行hexo s命令会启动Hexo本地服务器，以便你可以在本地计算机上预览你的博客。

​ 具体来说，该命令将在本地计算机上创建一个Web服务器，将生成好的静态网站文件作为该服务器的根目录，并监听指定的端口号，以允许你在本地计算机上通过浏览器来访问该服务器。

默认情况下，hexo s命令使用的端口号是4000，这意味着你可以通过访问”http://localhost:4000"来访问本地服务器。当然，如果你希望使用其他的端口号来启动服务器，也可以在命令中指定端口号，例如：

​ 其中，”-p”参数指定了要使用的端口号，这里是8888。这将启动一个监听8888端口的本地服务器，你可以在浏览器中通过”http://localhost:8888"来访问该服务器。

​ 需要注意的是，执行hexo s命令只是在本地计算机上启动了一个Web服务器，其他人是无法在互联网上访问到该服务器的。如果需要将你的博客发布到互联网上，需要使用hexo deploy等命令来将静态网站文件上传到远程服务器上。
